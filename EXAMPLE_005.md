Решение:
Ввести Execution Inbox как единственную точку входа
для всех решений, идей и запросов в ARCHITECTORIUM.

Действие:
Определён регламент Execution Inbox.

Правила:
— Inbox принимает всё без оценки.
— Inbox не является списком задач.
— Inbox не содержит решений.
— Inbox очищается только через разбор.

Процесс:
1. Любой входящий сигнал фиксируется в Inbox.
2. Сигнал маркируется (source / date).
3. На разборе принимается решение:
   — discard
   — defer
   — move to Execution Queue
   — transform into decision (EX-002)

Запреты:
— запрещено исполнять напрямую из Inbox
— запрещено обсуждать элементы Inbox
— запрещено хранить Inbox без регулярного разбора
